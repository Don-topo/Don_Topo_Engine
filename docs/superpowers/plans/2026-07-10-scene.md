# Scene Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Introducir una clase `Scene` que posea el árbol de GameObjects, encapsule la sincronización física↔transform por frame, y haga explícito (no dependiente del orden de declaración de variables) el shutdown ordenado respecto a `PhysicsManager`/`AudioManager`.

**Architecture:** `Scene` envuelve un `GameObject m_root` por valor (mismo patrón que el `root` local de hoy). Expone `addGameObject`/`removeGameObject`/`traverse`/`getRoot` para el árbol, `update(dt, physics)` para la sync física-transform (hoy dispersa en el loop de `main.cpp`), y `shutdown(physics, audio)` para la limpieza ordenada de componentes que dependen de managers externos (hoy un loop manual + comentario de advertencia en `main.cpp`). `Renderer`/`EditorUI` siguen usando `GameObject*` para dibujo (sin cambios de firma en el hot path de render); `EditorUI` gana un puntero no-propietario a `Scene` (mismo patrón que `m_physics`/`m_audio`) únicamente para poder llamar `removeGameObject` en el borrado diferido.

**Tech Stack:** C++20, CMake+Ninja, sin framework de tests unitarios (verificación por compilación + ejecución manual de `Sandbox.exe`).

## Global Constraints

- No hay gtest/ctest en el repo — cada tarea se verifica con `build.bat` (compila) y, en la tarea final, con una pasada manual de la app. Ver `configure.bat`/`build.bat` vía **PowerShell** (no Bash: son `.bat`, requieren `vcvarsall.bat` en el PATH del shell).
- `GameObject.h`/`.cpp` no se modifican — `parent` ya es público y suficiente para `removeGameObject`.
- No tocar `EditorUI::draw(...)` firma de dibujo ni el wiring `Renderer::m_sceneRoot` existente — solo añadir un puntero `Scene*` adicional, mismo patrón que `setPhysicsManager`/`setAudioManager`.
- Composición de componentes sigue siendo campos fijos en `GameObject` (no ECS) — fuera de scope.

### Task 3: Scene::update(dt, physics) — sync física↔transform

**Files:**
- Modify: `engine/src/Scene.cpp` (reemplazar el no-op de `update`)

**Interfaces:**
- Consumes: `GameObject::hasBoxCollider()/getBoxCollider()` y equivalentes Sphere/Capsule/Plane (`engine/include/DonTopo/GameObject.h:29-51`), `BoxCollider::isDynamic()/getWorldTransform()/syncTransform(...)` y equivalentes (patrón idéntico en los 4 tipos de collider).
- Produces: `Scene::update(float dt, PhysicsManager& physics)` — sincroniza cada collider con su `GameObject` y termina propagando `updateWorldTransforms()`.

- [ ] **Step 1: Implementar `Scene::update` en `engine/src/Scene.cpp`**

Reemplaza el no-op `void Scene::update(float, PhysicsManager&) {}` por (lógica calcada de `sandbox/src/main.cpp:214-261`, sin la parte de `liveCube`/renderer que se queda en `main.cpp`):

```cpp
void Scene::update(float /*dt*/, PhysicsManager& /*physics*/)
{
    m_root.traverse([](GameObject* go) {
        if (go->hasBoxCollider())
        {
            if (go->getBoxCollider()->isDynamic())
            {
                go->worldTransform = go->getBoxCollider()->getWorldTransform();
                glm::mat4 parentWorld = go->parent ? go->parent->worldTransform : glm::mat4(1.0f);
                go->localTransform = glm::inverse(parentWorld) * go->worldTransform;
            }
            else
                go->getBoxCollider()->syncTransform(go->worldTransform);
        }

        if (go->hasSphereCollider())
        {
            if (go->getSphereCollider()->isDynamic())
            {
                go->worldTransform = go->getSphereCollider()->getWorldTransform();
                glm::mat4 parentWorld = go->parent ? go->parent->worldTransform : glm::mat4(1.0f);
                go->localTransform = glm::inverse(parentWorld) * go->worldTransform;
            }
            else
                go->getSphereCollider()->syncTransform(go->worldTransform);
        }

        if (go->hasCapsuleCollider())
        {
            if (go->getCapsuleCollider()->isDynamic())
            {
                go->worldTransform = go->getCapsuleCollider()->getWorldTransform();
                glm::mat4 parentWorld = go->parent ? go->parent->worldTransform : glm::mat4(1.0f);
                go->localTransform = glm::inverse(parentWorld) * go->worldTransform;
            }
            else
                go->getCapsuleCollider()->syncTransform(go->worldTransform);
        }

        // Plane Collider siempre es kinematic (isDynamic()==false hardcoded):
        // nunca lee pose de PhysX, solo empuja la del GameObject.
        if (go->hasPlaneCollider())
            go->getPlaneCollider()->syncTransform(go->worldTransform);
    });

    m_root.updateWorldTransforms();
}
```

`dt` y `physics` quedan sin usar dentro del cuerpo (los colliders ya llevan su propio `PxActor` interno) — se mantienen en la firma tal como se aprobó en el spec, comentados con `/* */` para no generar ambigüedad sobre si se olvidó usarlos.
